# Deploy on Cloudflare

[简体中文](DeploymentZH)

## Prerequisites

- A Cloudflare account with Workers enabled and an active R2 subscription
- A GitHub account and a fork of `Leovikii/Sing-Sub`
- An administrator password containing at least 12 UTF-8 bytes

A private GitHub data repository and PAT are not required to deploy the site. They are only needed when GitHub synchronization or SRS compilation is enabled later.

## Connect the repository

1. Open **Cloudflare Dashboard > Workers & Pages** and create or select a Worker.
2. Open **Settings > Builds**, connect GitHub, and select your Sing-Sub fork.
3. Set the production branch to `main`.
4. Disable non-production branch builds unless you intentionally need previews.
5. Leave the root directory empty.
6. Set **Build command** to `npm run build`.
7. Set **Deploy command** to `npm run deploy:cloudflare`.
8. Use the Builds API token generated by Cloudflare.

## Set the administrator password

Add an encrypted build variable named `SING_SUB_ADMIN_PASSWORD`. Its value becomes the runtime `ADMIN_PASSWORD` during the first deployment and must contain at least 12 UTF-8 bytes.

Do not store this value in the repository, a plain build variable, a command argument, or a screenshot.

## First deployment

Connect the repository or retry the build to start deployment. The deployment initializer:

- creates or reuses the private R2 bucket `sing-sub-data`;
- preserves every existing object in that bucket;
- creates `ADMIN_PASSWORD` only when it is missing;
- independently generates missing `SESSION_SIGNING_SECRET` and `SUBSCRIPTION_SIGNING_SECRET` values;
- preserves existing runtime secrets during later deployments;
- deploys the Worker with the `WORKSPACE_BUCKET` binding.

Open the generated `workers.dev` URL and sign in with the administrator password. On an empty bucket, the first successful login creates an empty workspace and an editable Momo adapter.

After the first successful deployment, `SING_SUB_ADMIN_PASSWORD` may be removed from Build variables. The runtime `ADMIN_PASSWORD` remains in the Worker's encrypted variables. Restore the build secret before a later deployment only if the runtime administrator secret was deleted.

## Optional GitHub features

After login, open **Settings > Repository** to connect a private data repository and fine-grained PAT. This enables explicit import, push, pull, and optional SRS compilation. Normal WebUI editing and JSON subscriptions do not require GitHub.

When SRS is enabled, Sing Sub provisions the compiler workflow in the connected data repository. The workflow compiles rule sets and returns the result through a short-lived job ticket; it does not receive Cloudflare storage credentials.

## Updates

- The maintainer deployment tracks the official `main` branch.
- A user deployment tracks the `main` branch of that user's fork.
- To update a fork, use GitHub **Sync fork**. The resulting commit triggers Cloudflare Workers Builds.
- A failed build leaves the previously deployed Worker version active and does not modify R2 data.

## Recovery

Code and data recovery are separate operations:

- Revert a faulty source commit on `main` to produce a new Cloudflare build.
- Where the Cloudflare account exposes Worker version rollback, select a known-good version and redeploy it.
- Restore business data by publishing a selected immutable workspace revision as a new revision. Never clear R2 or move `head.json` backward manually.

The maintainer's production automatic update path is verified. A clean deployment under an independent Cloudflare account, the ordinary-user **Sync fork** path, and an actual Worker rollback remain deferred until a second account or a real incident provides a safe test target.

## Local verification

```powershell
npm ci
npm run verify
npm run worker:dry-run
```

Local deployment remains available to maintainers with `npm run deploy`, but it is not required for ordinary installation.

For the complete recovery model, see [release-and-recovery.md](https://github.com/Leovikii/Sing-Sub/blob/main/docs/operations/release-and-recovery.md).
